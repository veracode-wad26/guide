Getting Started

After forking a repo from "The Three Repos" above, clone and checkout the participant-start branch:

Step 1: Clone and checkout JavaScript:
git clone https://github.com/YOUR-USERNAME/verademo-javascript-api.git cd verademo-javascript-api && git checkout participant-start
Step 2: Clone and checkout Python:
git clone https://github.com/YOUR-USERNAME/verademo-python.git cd verademo-python && git checkout participant-start
Step 3: Clone and checkout Java:
git clone https://github.com/YOUR-USERNAME/verademo-java.git cd verademo-java && git checkout participant-start

Scanning Commands

Install OSV Scanner:
# macOS brew install osv-scanner # Windows winget install Google.OSVScanner
Run a scan:
osv-scanner scan -r . # For JSON output: osv-scanner scan -r -f json .

Manifest Files

Language Manifest Install Command
JavaScript package.json npm install

Fixing Vulnerabilities

Step 1: Identify vulnerable versions
Run OSV Scanner and note the CVEs and vulnerable package versions
Step 2: Update the manifest
Bump each vulnerable package to the safe version listed in the scan output
Step 3: Verify locally
Run tests to ensure the app still works after updates
Step 4: Push your changes
Commit and push — GitHub Actions will scan and update the leaderboard

Common Issues

OSV Scanner shows findings I didn't fix
Check for transitive dependencies. A package you kept may pull in a vulnerable version indirectly. Look at the full audit output and bump the parent package version.
Tests fail after updating dependencies
A major version bump may have breaking changes. Try a minor version bump first, or check the package changelog for migration guidance.
Package not found during install
Make sure you're using the correct package name and version. Double-check the manifest for typos or version constraints.
GitHub Actions scan is still running
Check the Actions tab in your fork. If it's stuck, try pushing another commit or check for errors in the workflow logs.
